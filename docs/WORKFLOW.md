# آلية العمل التعاوني - TurkTextileHub Bot

## الفريق

| الدور | المسؤولية |
|---|---|
| **مساعد الإدارة (أنا)** | تخطيط المشروع، إعداد المواصفات، مراجعة الكود، النشر، إدارة GitHub |
| **Claude / ChatGPT / Gemini** | كتابة الكود بناءً على المواصفات |
| **أبو فايز** | التنسيق بين الفريق، اتخاذ القرارات، اختبار البوت |

---

## دورة العمل لكل مرحلة

```
1. أنا أُعدّ ملف مواصفات المرحلة (PHASE_X_REQUIREMENTS.md)
         ↓
2. أبو فايز يأخذ الملف ويعطيه لـ Claude/ChatGPT/Gemini
         ↓
3. النموذج يكتب الكود ويسلّمه لأبو فايز
         ↓
4. أبو فايز يرسل الكود لي
         ↓
5. أنا أراجع الكود وأدمجه في المشروع
         ↓
6. أنا أرفع الكود على GitHub
         ↓
7. الاختبار والانتقال للمرحلة التالية
```

---

## المراحل الكاملة للمشروع

| المرحلة | الوصف | الحالة |
|---|---|---|

| **المرحلة 2** | تسجيل الموردين (conversation handler متعدد الخطوات) | ⏳ قادمة |
| **المرحلة 3** | خدمة قاعدة البيانات (Supabase HTTP integration) | ⏳ قادمة |
| **المرحلة 4** | تسجيل التجار | ⏳ قادمة |
| **المرحلة 5** | لوحة الإدارة وعرض المنتجات | ⏳ قادمة |
| **المرحلة 6** | النشر النهائي والتوثيق | ⏳ قادمة |

## تعليمات لأبو فايز عند إعطاء المواصفات للنماذج

### الرسالة الافتتاحية الموصى بها:
```
أنا أبني بوت تليجرام B2B لتجارة المنسوجات التركية.
لدي مواصفات تفصيلية للمرحلة الحالية.
أريدك أن تكتب الكود بالكامل بناءً على هذه المواصفات.
يجب أن يكون الكود نظيفاً مع تعليقات بالعربية.
لا تضيف أي ميزات غير مذكورة في المواصفات.

[ثم تلصق محتوى ملف المواصفات]
```

### عند استلام الكود:
- تأكد أن النموذج أعطاك **جميع الملفات** المذكورة في المواصفات
- إذا نسي ملفاً، اطلب منه تحديداً: "أعطني محتوى ملف X"
- أرسل لي الكود كاملاً في رسالة واحدة أو ملف
