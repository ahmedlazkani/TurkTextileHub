السلام عليكم فريق Kayisoft،

يعطيكم العافية على الشغل الممتاز.
نحن الآن في مرحلة بناء "البوست الاحترافي" الذي يُنشر تلقائياً على قناة المورد بعد إضافة المنتج.

لكي يكون البوست سوبر احترافي ويأخذ الزبون مباشرة إلى التطبيق (TopGate)، سنضيف زرين لكل بوست:
1. **[💬 تواصل مع المورد]** → يأخذ الزبون لشات مع المورد حول هذا المنتج تحديداً.
2. **[🛍️ صفحة المنتج]** → يأخذ الزبون لصفحة المنتج داخل التطبيق.

**المشكلة الحالية:**
بعد أن نقوم بإنشاء المنتج عبر `POST api/seller/products`، نحتاج إلى `product_id` و `seller_id` و `seller_name` لبناء الروابط الذكية (Smart Links) لهذين الزرين.

**المطلوب (Endpoint جديد):**
نحتاج Endpoint بسيط يجلب تفاصيل المنتج بعد إنشائه:

```http
GET api/seller/products/{product_id}
```

**الـ Response المتوقع:**
```json
{
  "id": "uuid",
  "name": "string",
  "seller_id": "uuid",
  "seller_name": "string",
  "category_id": "uuid"
}
```

هذا سيسمح لنا ببناء الروابط الذكية (Deferred Deep Links) التي طلبناها سابقاً، بحيث إذا لم يكن التطبيق موجوداً عند الزبون، يأخذه للستور، وبعد التحميل يفتح له صفحة المنتج أو الشات مباشرة.

بانتظار تأكيدكم لدمج هذا الـ Endpoint في البوت.
